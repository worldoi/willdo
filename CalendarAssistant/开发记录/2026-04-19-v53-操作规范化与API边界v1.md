# 2026-04-19 v53 操作规范化与 API 边界 v1

## 1. 目标

- 统一项目内“操作”的归类与调用路径。
- 约束 center-port-node 的适用范围，避免 UI 瞬态状态污染领域层。
- 为后续 API 化改造提供可执行清单。

## 2. 术语与分类

### 2.1 前端操作（UI 操作）

- 仅改变界面状态，不改变业务事实。
- 不落库，不触发通知/同步/系统日历等外部副作用。
- 允许写在调用页面/组件内部。

### 2.2 后端操作（Domain/Data 操作）

- 改变业务事实（事件、课程、设置、归档、同步状态等）。
- 需要持久化，或触发外部副作用（通知调度、系统日历、Worker、Receiver）。
- 必须走 center-port-node API。

### 2.3 混合操作（UI 触发 + Domain 提交）

- 入口在 UI，最终必须调用后端 API。
- UI 只负责收集参数、展示结果，不直接改 Repository/DAO。

## 3. 判定规则

1. 写 Repository/DAO -> 后端操作。
2. 触发通知、同步、系统日历、Worker、Receiver 链路 -> 后端操作。
3. 仅焦点、手势、动画、弹层、搜索框文本 -> 前端操作。
4. 前端内部代码禁止直接触发外部副作用。
5. 前端状态若需要跨页面共享、可回放或持久化，升级为后端操作。

## 4. 当前项目操作归类（v1）

## 4.1 后端操作（必须 API 化）

### 日程域

- 新增日程
- 编辑日程
- 删除日程
- 重要标记切换
- 规则动作（完成/检票/撤销）
- 重复日程实例操作（含 this/future/all 语义）

### 课程域

- 新增课程
- 编辑课程
- 删除课程
- 单次课程排除/改单次实例

### 归档域

- 归档
- 恢复
- 删除归档条目
- 清空归档
- 自动归档过期

### 设置域

- AI 配置更新（文本/多模态）
- 天气配置更新
- 主题与 UI 尺寸更新
- 偏好项更新（通知、胶囊、短信监控、便签开关等）
- 作息/学期配置更新

### 同步与副作用域

- 日历同步开关
- 立即同步
- 手动同步
- 同步来源日历更新
- 反向同步触发
- 提醒调度/取消调度

### 导入导出与识别入库

- 课程/日程导入导出
- WakeUp 导入
- Prompt 导入导出与版本更新确认
- OCR/AI 识别后入库
- 短信三通道入库（广播/通知监听/内容观察）

## 4.2 前端操作（允许页面内实现）

- 导航与路由切换（Tab、侧边栏、设置子页）
- 搜索栏展开/收起与搜索词输入
- 弹窗、Sheet、遮罩显隐
- 焦点与键盘控制
- 手势中间态与动画进度
- 列表滚动位置与局部展开态
- Toast/Snackbar/Loading 展示状态

## 5. All 页面展示规则（本次定稿）

适用文件：`app/src/main/java/com/antgskds/calendarassistant/ui/page_display/AllEventsPage.kt`

- 不展示便签（`tag != NOTE`）。
- 不展示重复父事件（`!isRecurringParent`）。
- 展示重复子实例与普通事件。
- 统一限制展示窗口：仅显示 `startDate <= today + 7 天` 的事件。
- 搜索、排序逻辑在过滤后的集合上执行。

## 6. API 化落地顺序（建议）

### P0（先做）

- 日程域操作 API 收口（增删改、完成/检票/撤销、归档/恢复）。
- 统一副作用出口（提醒、胶囊、同步）。

### P1（随后）

- 课程域与重复实例 1~6 语义 API 化。
- 导入导出与识别入库 API 化。

### P2（最后）

- 设置域 API 统一签名与审计字段（来源、traceId、时间戳）。
- Receiver/Tile/Floating 入口全部走 API。

## 7. 强约束

- 前端文件不直接写 DAO/Repository。
- 外部副作用只能由后端 API 链路触发。
- 操作语义以统一 API 为准，页面仅负责触发与展示。
