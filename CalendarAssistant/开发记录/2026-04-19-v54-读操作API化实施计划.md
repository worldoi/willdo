# 2026-04-19 v54 读操作 API 化实施计划

## 1. 背景

当前项目写链路已基本收口为：

调用方 -> `*OperationApi` -> `AppRepository`

但读链路仍存在直接依赖 `AppRepository` 的情况（UI 页面、ViewModel、部分系统入口），前后端职责边界仍不彻底。

## 2. 目标

- 页面与 ViewModel 的业务读操作统一走 `QueryApi`。
- UI 层不再直接 `AppRepository.getInstance(...)`。
- 保持现有业务行为不变（本次为结构性改造，不改语义）。

## 3. 边界与原则

### 3.1 纳入 API 化（必须）

- 业务数据读取：events/courses/settings/archivedEvents、同步状态、可选日历、统计计数等。

### 3.2 不纳入 API 化（保留页面内）

- 焦点、键盘、手势中间态、动画、滚动位置、弹层开关、输入草稿等 UI 瞬态。

### 3.3 展示过滤归属

- 展示过滤仍在前端页面/VM 处理（例如 All 页未来 7 天规则）。

## 4. 现状问题点（已识别）

- 页面直连仓储：
  - `ui/page_display/settings/PreferenceSettingsPage.kt` 使用 `AppRepository.getInstance(...)` 做读取判断。
- ViewModel 读仓储较重：
  - `MainViewModel` 直接组合 `repository.events/courses/settings`。
  - `SettingsViewModel` 直接暴露 `repository.settings` 并直接调用计数读取。
- 结果：读写分层不对称，center-port-node 在读侧未闭环。

## 5. 方案设计

### 5.1 新增读接口（core）

- `core/query/ScheduleQueryApi.kt`
- `core/query/SettingsQueryApi.kt`

建议能力（首版）：

`ScheduleQueryApi`
- `eventsFlow`
- `coursesFlow`
- `archivedEventsFlow`
- `findEventById(...)`（可选）
- `fetchArchivedEvents()`
- `getEventsCount()/getTotalEventsCount()/getCoursesCount()`

`SettingsQueryApi`
- `settingsFlow`
- `getSyncStatus()`
- `getSelectableSyncCalendars()`

### 5.2 新增仓储适配（data）

- `data/query/AppRepositoryScheduleQueryApi.kt`
- `data/query/AppRepositorySettingsQueryApi.kt`

职责：仅做 `QueryApi -> AppRepository` 读适配，不承载业务语义。

### 5.3 应用注入

在 `App.kt` 增加：

- `scheduleQueryApi`
- `settingsQueryApi`

与现有 `scheduleOperationApi/settingsOperationApi` 并行。

### 5.4 调用侧迁移顺序

1. 先迁 ViewModel（低风险）
   - `MainViewModel` 改读 `ScheduleQueryApi/SettingsQueryApi`
   - `SettingsViewModel` 改读 `SettingsQueryApi`
2. 再迁页面直连仓储点
   - `PreferenceSettingsPage.kt` 去掉 `AppRepository.getInstance(...)`
3. 最后做约束
   - UI 页面层禁止直接依赖 `AppRepository`

## 6. 分阶段实施计划（明日执行）

### P0 - 骨架搭建
- 新建 QueryApi 接口与适配层
- App 注入 query api

### P1 - VM 迁移
- MainViewModel 读链路迁移
- SettingsViewModel 读链路迁移

### P2 - 页面清理
- PreferenceSettingsPage 去仓储直连
- 由 ViewModel 暴露所需查询能力

### P3 - 收口
- 检索确认 UI 页面无 AppRepository 直连
- 保持行为一致后移除冗余读取路径

## 7. 验收标准

- `ui/page_display` 无 `AppRepository.getInstance(...)`
- VM 主读链路不直接访问 `repository.events/settings/courses`
- 编译通过 + 安装通过
- 回归通过：
  - All 页展示规则不变
  - 设置页同步状态显示正常
  - 提前提醒冲突提示逻辑正常

## 8. 风险与回滚

### 风险
- 读源迁移中可能出现状态不同步或刷新时机变化。
- API 注入参数增多，构造器易混乱。

### 回滚策略
- 按阶段提交（P0/P1/P2）
- 任一阶段异常可单独回退，不影响写链路 API 化成果。
